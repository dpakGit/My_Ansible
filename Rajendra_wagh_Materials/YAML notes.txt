1Ô∏è‚É£ What Is YAML?
YAML (Yet Another Markup Language) is a human-readable data serialization format.
Designed for configuration files, data storage, and data exchange.

2Ô∏è‚É£ Purpose of Using YAML
Represent data in key-value pairs.
Define configuration files (Ansible, Kubernetes, Docker Compose).
Share data between programs/services.
Supports complex data structures (lists, dictionaries).

3Ô∏è‚É£ Where to Use YAML?
DevOps & Automation: Ansible playbooks, Kubernetes manifests
CI/CD Pipelines: GitHub Actions, GitLab CI
Configuration files: Docker Compose, CircleCI, TravisCI
Data Serialization: Store structured data in readable format

6Ô∏è‚É£ Quick Check

YAML uses indentation, not braces.
Supports lists (- item) and maps (key: value).
Human-friendly & easy to read.

7Ô∏è‚É£ Basic Rules for YAML Syntax

Use spaces, not tabs (2 spaces per level recommended).
Key-value separator is : followed by a space.
Lists start with - followed by a space.
Strings with special characters can be quoted.

8Ô∏è‚É£ Key Concepts in YAML Syntax
Scalars: single values (string, number, boolean)
Lists/Arrays: - item1
Mappings/Objects: key: value
Nested structures: indentation defines hierarchy

Block Folded Style >

1Ô∏è‚É£ Anchors (&) in YAML
Anchors let you define a piece of data once and reuse it later.

defaults: &default_settings
  user: raj
  shell: /bin/bash
  home: /home/raj

&default_settings ‚Üí anchor name

2Ô∏è‚É£ Aliases (*) in YAML

Aliases let you reference data defined with an anchor.
admin:
  <<: *default_settings
  role: admin

*default_settings ‚Üí references the anchor

<<: ‚Üí merges the referenced map into the new map

Resulting data for admin:

user: raj
shell: /bin/bash
home: /home/raj
role: admin

3Ô∏è‚É£ References in YAML Configuration

Anchors and aliases are collectively called references.
Avoid repeating same data in multiple places.

Example:

webserver: &web
  ip: 192.168.1.10
  name: Raj-Web

dbserver:
  <<: *web
  name: Raj-DB


Output:

dbserver:
  ip: 192.168.1.10
  name: Raj-DB

4Ô∏è‚É£ When to Use References?

Reuse common variables or configs
Avoid duplicate entries in large YAML files
Maintain consistency across multiple hosts/services

5Ô∏è‚É£ What Are YAML Tags?

Tags define the data type of a value explicitly.
Syntax:
!tag value

Example: !str, !int, !bool

6Ô∏è‚É£ Setting a Custom URI

You can define a custom tag URI:

!mytag:yaml.org,2002:str "Hello Raj"

!mytag:uri ‚Üí user-defined tag

yaml.org,2002: ‚Üí namespace for standard YAML types
str ‚Üí type (string)
Other examples of standard YAML 1.1 types:
yaml.org,2002:int ‚Üí integer
yaml.org,2002:bool ‚Üí boolean
yaml.org,2002:float ‚Üí float
Used in advanced YAML parsing

7Ô∏è‚É£ Setting a Local Tag

Local tags are shorter, relative tags:

message: !raj "This is a local tag example"

Easier to read than full URI

Mostly for internal use in YAML parser

8Ô∏è‚É£ Setting Data Type in YAML
Type				Example
String			name: "Raj"
Integer			age: !int 35
Float				pi: !float 3.14
Boolean			active: !bool true
Null				notes: !null ~
Date				dob: !date 1990-01-01

Example using tags:

name: !str Raj
age: !int 35

‚úÖ Summary

Anchors (&) ‚Üí define reusable data

Tags (!tag) ‚Üí explicitly define data type

Custom/Local tags ‚Üí advanced type or domain-specific usage

1Ô∏è‚É£ YAML File Operations
You can read, write, and validate YAML files in most programming languages.
Loading ‚Üí read YAML data into memory
Saving/Dumping ‚Üí write YAML data back to a file
Validation ‚Üí ensure syntax and types are correct

2Ô∏è‚É£ Loading YAML Data
ex:Python
import yaml
with open('data.yaml', 'r') as file:
    data = yaml.safe_load(file)
print(data['name'])  # Raj

ex:JavaScript (Node.js)
const fs = require('fs');
const yaml = require('js-yaml');
let fileContents = fs.readFileSync('data.yaml', 'utf8');
let data = yaml.load(fileContents);
console.log(data.name); // Raj

ex:Java
import org.yaml.snakeyaml.Yaml;
import java.io.FileInputStream;
import java.util.Map;
Yaml yaml = new Yaml();
FileInputStream input = new FileInputStream("data.yaml");
Map<String, Object> data = yaml.load(input);
System.out.println(data.get("name")); // Raj

3Ô∏è‚É£ Saving YAML Data
ex:Python
import yaml
data = {'name': 'Raj', 'age': 35}
with open('output.yaml', 'w') as file:
    yaml.dump(data, file)

ex:JavaScript
const fs = require('fs');
const yaml = require('js-yaml');
let data = {name: 'Raj', age: 35};
fs.writeFileSync('output.yaml', yaml.dump(data));

ex:Java
import org.yaml.snakeyaml.Yaml;
import java.io.FileWriter;
Yaml yaml = new Yaml();
Map<String, Object> data = Map.of("name", "Raj", "age", 35);
FileWriter writer = new FileWriter("output.yaml");
yaml.dump(data, writer);

4Ô∏è‚É£ Validation and Error Handling
Importance:
Avoid syntax errors in YAML that break automation scripts (Ansible, Kubernetes, etc.)
Ensure correct data types
Techniques:
Use language-specific YAML parser exceptions (yaml.YAMLError in Python)
Use online YAML validators: https://yamlchecker.com
Linting tools: yamllint

Example (Python):
import yaml
try:
    with open('data.yaml') as file:
        data = yaml.safe_load(file)
except yaml.YAMLError as e:
    print("Error in YAML file:", e)
