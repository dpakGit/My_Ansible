⭐ ANSIBLE — ALL LEVELS (Basic → Intermediate → Expert)

✅ LEVEL-1 (Basic) — Ansible

1. What is Ansible?

An automation tool for configuration management, application deployment, and orchestration. Agentless and uses SSH/WinRM.

2. What is a Playbook?

A YAML file that contains tasks to execute on remote machines.

3. What does “agentless” mean?

No software is installed on client nodes; Ansible uses SSH/WinRM.

4. How do you check connectivity to all hosts?
ansible all -m ping

5. What is Inventory?

A file containing host IPs, groups, and variables.

6. What is Idempotency?

Running the same play multiple times results in the same state.

7. What are Ansible Modules?

Small programs like yum, apt, copy, template, service.

8. What is Ansible Galaxy?

Public repository of roles.

9. What is Ansible Vault?

Used to encrypt sensitive information.

10. What is a Task?

A single automation step inside a play.

🟦 LEVEL-2 (Intermediate) — Ansible
1. What are Facts in Ansible?

System information gathered from nodes.

ansible all -m setup

2. What are Handlers?

Triggered only when notified by tasks (example: restart nginx).

3. What is a Role?

A structured way to organize tasks, handlers, templates, defaults, files, vars.

4. What is the difference between copy and template?

copy → static files

template → uses Jinja2 variables

5. What are Conditionals?

Used for task execution logic:

when: ansible_os_family == "Debian"

6. What are Loops?

Execute the same task for multiple items:

loop:
  - git
  - nginx

7. How do you pass variables from CLI?
ansible-playbook app.yml -e "version=2.0"

8. What is Dynamic Inventory?

Inventory generated automatically from cloud API (AWS, Azure, GCP).

9. How to run only specific tags?
ansible-playbook site.yml --tags "install"

10. What is check mode?

Dry run:

ansible-playbook site.yml --check

🔥 LEVEL-3 (Expert) — Ansible

2. What is Ansible Tower/AWX?

A web UI + RBAC + Reports + Scheduling on top of Ansible.

3. How do you implement Zero-Downtime deployments with Ansible?

Use rolling updates

Use serial: 1 or serial: 10%

Integrate with load balancers

4. How do you manage secrets across environments?

Vault

HashiCorp Vault

AWS SSM/Secrets Manager

Environment variables

5. How do you optimize Ansible performance?

pipelining = True

Disable fact gathering when not required

Use async tasks

SSH multiplexing

Limit tasks using tags

6. What is Delegation in Ansible?

Run a task on a different host:

delegate_to: 127.0.0.1

7. What is Ansible Collections?

A packaging format for sharing plugins, roles, modules.

8. How do you handle configuration drift?

Run:

--check

Scheduled Tower/AWX jobs

GitOps approach

9. How do you integrate Ansible with CI/CD?

Jenkins pipeline → run Ansible playbooks

GitHub Actions → deploy with Ansible

Use ansible-lint + molecule

10. Why is Ansible preferred over Chef/Puppet?

Agentless

Simple YAML

Easy learning curve

No master/slave architecture

Fast deployment

⭐ TERRAFORM — ALL LEVELS (Basic → Intermediate → Expert)
✅ LEVEL-1 (Basic) — Terraform
1. What is Terraform?

IaC tool for provisioning cloud infrastructure.

2. What is HCL?

HashiCorp Configuration Language.

3. What does terraform init do?

Downloads provider plugins and initializes backend.

4. What does terraform plan do?

Shows what changes will be applied.

5. What is terraform apply?

Executes infra changes.

6. What is a Resource?

Basic building block for cloud components.

7. What is a Provider?

Plugin to connect Terraform to AWS/Azure/GCP.

8. What is a Variable?

Input to Terraform.

9. What is an Output?

Printed result after apply.

10. What is Terraform State File?

A JSON file that stores the real infrastructure state.

🟧 LEVEL-2 (Intermediate) — Terraform
1. What is Remote Backend?

Stores state remotely (S3, GCS, Terraform Cloud).

2. Why is state locking needed?

Prevents simultaneous modifications.

3. What is a Terraform Module?

Reusable collection of resources.

4. What is terraform import?

Brings existing resources under Terraform management.

5. What are Data Sources?

Read-only lookup values.

6. What is the depends_on meta-argument?

Manually enforce dependencies.

7. What is locals?

Reusable local expressions.

8. What is terraform refresh?

Updates state with real infrastructure.

9. How to override variable values?

CLI -var

.tfvars file

Environment variables

10. What is Drift Detection?

Comparing actual infra vs Terraform state.
